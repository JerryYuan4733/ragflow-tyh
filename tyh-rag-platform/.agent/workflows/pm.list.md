## User Input

```text
$ARGUMENTS
```

You **MUST** consider the user input before proceeding (if not empty).

## 流程概述

本工作流用于**查看所有迭代、需求文档和原型的状态**。

---

### 1. 扫描目录

扫描 `docs/prototypes/` 目录，获取所有迭代和文件。

### 3. 文件状态判断逻辑

| 状态 | 判断条件 |
|------|----------|
| ✅ 澄清完成 | 存在 `{功能名}-澄清.md` |
| ✅ 需求整理 | 存在 `{功能名}-需求.md` |
| ✅ 原型完成 | 存在 `{功能名}-prototype.html` |
| ⏳ 进行中 | 文件不存在 |
| 📦 已归档 | 迭代 README.md 中标记为已归档 |

---

## 筛选选项

用户可以在 $ARGUMENTS 中指定筛选条件：

```bash
# 只看某个迭代
/pm.list 迭代:2026-01-用户管理

# 只看未完成的
/pm.list 状态:进行中

# 只看原型
/pm.list 类型:原型
```

---

## Key Rules

- **实时扫描**：每次执行都重新扫描目录
- **状态准确**：基于文件存在性判断状态
- **格式清晰**：使用树形结构展示

## 使用示例

```bash
# 查看所有
/pm.list

# 筛选迭代
/pm.list 2026-01-用户管理

# 筛选状态
/pm.list 进行中
```
