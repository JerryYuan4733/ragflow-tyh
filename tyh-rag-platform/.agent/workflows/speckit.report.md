## 流程概述

本工作流对应开发流程 **⑩ 全流程开发报告**，产出物为 `{功能}-开发报告.md`，归档至 `docs/report/`。

### 步骤

1. **识别当前功能**：
   - 获取当前分支名
   - 在 `.specify/specs/` 中找到对应文档

2. **加载上下文**：
   - 读取所有阶段文档：
     - `spec.md` - 需求规格
     - `product.md` - 产品设计
     - `architecture.md` - 架构设计
     - `plan.md` - 开发计划
     - `tasks.md` - 任务清单
     - `testcase.md` - 测试用例
     - `testreport.md` - 测试报告
   - 读取 Git 提交记录
   - 读取 `.specify/memory/constitution.md`

3. **汇总分析**：
   - 统计开发周期
   - 计算计划偏差
   - 整理技术决策
   - 总结经验教训

4. **生成开发报告**：
   
   ### 报告内容
   - **项目概述**：功能描述、目标达成
   - **开发过程**：各阶段耗时、关键里程碑
   - **技术总结**：架构决策、技术挑战
   - **质量报告**：测试覆盖、缺陷统计
   - **经验沉淀**：最佳实践、改进建议

5. **保存并归档**：
   - 写入 `.specify/specs/<feature>/report.md`（工作副本）
   - 归档到 `docs/report/{功能}-开发报告.md`

6. **报告**：输出开发报告路径、关键指标
